Child Flows

Tribe gives us a neat way of expressing alternate flows.

Our flow now encapsulates the entire navigation flow for the application proces, started in the host pane.

The new startBusiness and startPersonal messages, published from the welcome pane, trigger alternate child flows within our main flow. By default, these child flows will be ended whenever messages are handled in the main flow.

We could also have modelled our main navigation flow this way and kept our child flows separate by using the start flow helper:

'startBusiness': flow.start(BusinessFlow, details)

That's it for now! For more information, check out the Saga and Flow reference pages.
